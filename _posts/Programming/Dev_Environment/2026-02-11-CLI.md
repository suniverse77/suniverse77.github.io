우리는 일반적으로 터미널에서 Bash shell을 이용해 리눅스 OS와 소통한다.
<br>
보통은 리눅스 환경에서 터미널에서 치는 모든 명령어를 묶어서 리눅스 명령어라고 부르기는 하지만, 엄밀히 말하면 Bash 명령어와 리눅스 명령어는 다르다.

**Bash 명령어**는 shell이라는 프로그램 자체가 원래부터 할 줄 아는 내장 명령어이다. 반면, **리눅스 명령어**는 shell이 처리할 수 없는 복잡한 일들을 처리하기 위해 리눅스 OS 내부에 따로 설치해 둔 별도의 외부 프로그램들이다.

내가 지금 입력하는 명령어가 Bash 명령어인지, 리눅스 명령어인지 궁금하다면 터미널에 `type {명령어}`를 입력해서 확인할 수 있다.

```bash
user@server:~$ type cd
cd is a shell builtin
user@server:~$ type cat
cat is /usr/bin/cat
```

find
이름 그대로 컴퓨터 안에서 파일이나 폴더를 검색할 때 씁니다. grep이 텍스트 파일 '안의 내용'을 찾는다면, find는 조건(이름, 확장자, 생성 날짜 등)에 맞는 파일 '자체'를 찾아냅니다.

top
윈도우의 '작업 관리자'나 Mac의 '활성 상태 보기'처럼, 현재 시스템의 CPU 및 메모리 사용량과 실행 중인 프로그램(프로세스) 목록을 실시간 화면으로 보여줍니다.

df
(Disk Free의 약자) 현재 시스템에 연결된 하드디스크나 SSD의 전체 용량과 남은 용량을 직관적으로 확인할 수 있게 해줍니다. 주로 df -h 형태로 사용하여 사람이 읽기 쉬운 단위(GB, MB)로 봅니다.

tar
여러 개의 파일이나 폴더를 하나의 파일로 묶거나(아카이빙), 묶인 파일을 다시 풀 때 사용하는 명령어입니다. 리눅스 환경에서 무언가를 백업하거나 배포할 때 숨 쉬듯이 사용됩니다.

chmod
(Change Mode의 약자) 파일이나 폴더의 접근 권한(읽기, 쓰기, 실행)을 변경합니다. 인터넷에서 다운로드한 스크립트를 실행하려는데 "권한이 거부되었습니다"라는 에러가 뜰 때 이 명령어로 해결합니다.

## Bash 명령어

echo
화면에 텍스트나 변수 값을 출력합니다. 쉘 스크립트를 작성할 때 진행 상황을 알림 메시지처럼 띄우거나, 파일에 특정 문구를 덮어쓸 때 매우 자주 쓰입니다.

history
지금까지 내가 터미널에 입력했던 명령어들의 기록을 번호와 함께 순서대로 보여줍니다. 예전에 길게 타이핑했던 복잡한 명령어가 기억나지 않을 때 구세주 같은 역할을 합니다.

alias
자주 쓰는 길고 복잡한 명령어에 나만의 짧은 별명을 붙여주는 '단축키 설정' 기능입니다. 예를 들어 alias c="clear"라고 설정하면, 터미널에 c만 쳐도 화면이 깨끗하게 지워집니다.

export
운영체제에서 공통으로 사용할 '환경 변수'를 설정할 때 사용합니다. 새로운 프로그램을 설치하고 그 프로그램의 실행 경로를 시스템에 알려줄 때 필수적으로 쓰입니다.
