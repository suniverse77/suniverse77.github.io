네트워크, 호스트, IP, 포트는 동네와 건물에 비유해서 쉽게 이해할 수 있다.

- **네트워크:** 건물이 모여 있는 동네
- **호스트:** 동네에 세워진 건물
- **IP:** 건물을 찾아가기 위한 주소
- **포트:** 건물 내에 있는 각 방의 번호

![fig1](Programming/Dev_Environment/Host_IP_Port-1.png){: style="display:block; margin:0 auto; width:80%;"}

### 호스트 (Host)

호스트는 네트워크에 연결되어 데이터를 주고받을 수 있는 기기 그 자체를 의미한다.
<br>
예를 들어 노트북, GPU 서버, 스마트폰 모두 각각이 하나의 호스트가 된다.

### IP (Internet Protocol)

IP는 호스트가 네트워크에서 데이터를 주고받기 위해 부여받은 고유한 주소다.

서버를 띄울 때, 접근 권한을 설정하는 용도로 아래 두 IP를 자주 만나게 된다.

- **`127.0.0.1` (Localhost)** 

    네트워크 상에서 자신을 가리키는 특별한 주소로, 내 컴퓨터에서 보내는 요청만 허용할 때 사용한다.
- **`0.0.0.0`** 

    모든 IP를 통해 들어오는 외부 접속을 허용할 때 사용한다.

### 포트 (Port)

포트는 IP 주소를 통해 특정 호스트까지 무사히 도착한 데이터가, 그 호스트 안에서 실행 중인 여러 프로그램 중 어떤 프로그램과 통신할지 정해주는 출입구 번호다.

컴퓨터 안에서는 웹 브라우저, 메신저, AI 서버 등 수많은 프로그램이 동시에 돌아가고 있기 때문에, 데이터가 엉뚱한 곳으로 가지 않도록 포트 번호로 목적지를 확실히 구분해 주어야 한다.

포트 번호는 `0`번부터 `65535`번까지 총 65,536개가 존재한다.
<br>
마음대로 아무 번호나 써도 통신은 가능하지만, 전 세계 개발자들은 특정 프로그램에는 특정 포트를 쓰자고 사회적 약속을 정해두었다.
<br>
특히 `0`번부터 `1023`번까지는 시스템 운영을 위해 전 세계적으로 엄격하게 용도가 정해져 있으며, 이를 Well-Known Ports라고 부른다.

- **`80` (HTTP):** 인터넷에서 웹 페이지를 주고받을 때 사용하는 기본 포트
- **`22` (SSH):** 외부에서 서버 컴퓨터에 안전하게 원격 접속할 때 사용하는 포트
- **`8000` (FastAPI):** AI 서빙 API를 띄울 때 기본으로 잡히는 포트
