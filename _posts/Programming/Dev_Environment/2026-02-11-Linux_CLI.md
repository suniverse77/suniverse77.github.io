우리는 일반적으로 터미널에서 Bash shell을 이용해 리눅스 OS와 소통한다.
<br>
보통은 리눅스 환경에서 터미널에서 치는 모든 명령어를 묶어서 리눅스 명령어라고 부르기는 하지만, 엄밀히 말하면 Bash 명령어와 리눅스 명령어는 다르다.

**Bash 명령어**는 shell이라는 프로그램 자체가 원래부터 할 줄 아는 내장 명령어이다. 반면, **리눅스 명령어**는 shell이 처리할 수 없는 복잡한 일들을 처리하기 위해 리눅스 OS 내부에 따로 설치해 둔 별도의 외부 프로그램들이다.

내가 지금 입력하는 명령어가 Bash 명령어인지, 리눅스 명령어인지 궁금하다면 터미널에 `type {명령어}`를 입력해서 확인할 수 있다.

```bash
user@server:~$ type cd
cd is a shell builtin
user@server:~$ type cat
cat is /usr/bin/cat
```

아래는 우리가 주로 리눅스 환경에서 사용하는 CLI 명령어들을 정리한 것이다.

### 파일 및 디렉토리 (폴더) 관리

| 명령어 | 설명 | 사용 예시 |
| :---: | :---: | :---: |
| `ls` | 현재 위치에 있는 파일과 폴더 목록 출력 (List) | `ls -al`을 사용하면 숨김 파일까지 확인할 수 있다. |
| `cd` | 다른 폴더로 이동 (Change Directory) | 상위 폴더로 이동: `cd ..` <br> 홈 디렉토리로 이동: `cd ~` |
| `pwd` | 현재 내가 있는 위치의 절대 경로 출력 (Print Working Directory) |
| `mkdir` | 새로운 폴더 생성 (Make Directory) |
| `rm` | 파일이나 폴더 삭제 (Remove) | 파일 삭제: `rm {파일명}` <br> 폴더 삭제: `rm -rf {폴더명}` |
| `cp` | 파일이나 폴더 복사 (Copy) | 같은 폴더에 이름만 바꿔서 복사: `cp {복사할 파일} {새로 만들 파일}` <br> 파일을 다른 폴더로 복사: `cp {복사할 파일} {저장할 폴더}` <br> 폴더를 통째로 복사: `cp -r {복사할 폴더} {새로 만들 폴더}` |
| `mv` | 파일을 다른 곳으로 이동 (Move) |

- `ls -al`을 사용하면 숨김 파일까지 확인할 수 있다.
- `cd ..`는 상위 폴더, `cd ~`는 홈 디렉토리로 이동한다.
- 파일을 삭제할 때는 `rm {파일명}`, 폴더를 삭제할 때는 `rm -rf {폴더명}` 형태로 사용한다.

| `tail` | 파일의 맨 뒷부분만 출력 |  |
| `nano` |  |  |

### 파일 검색 및 데이터 필터링

| 명령어 | 설명 | 사용 예시 |
| :---: | :---: | :---: |
| `grep` | 텍스트 파일 안에서 특정 단어가 포함된 줄을 찾아줌 | `grep {찾을 단어} {파일명}` |
| `find` | 컴퓨터 안에서 특정 이름이나 확장자를 가진 파일/폴더를 찾아 | 

### 시스템 및 프로세스 관리

| 명령어 | 설명 | 예시 |
| :---: | :---: | :---: |
| `ps` | 현재 실행 중인 프로세스들의 목록 출력 (Process Status) |  |
| `kill` | 프로그램이나 프로세스를 강제 종료 |  |
| `df` | 하드디스크나 SSD의 전체 용량과 남은 여유 공간 출력 (Disk Free) |  |
| `du` | 특정 폴더나 파일이 디스크 용량을 얼마나 차지하는지 출력 (Disk Usage) |  |

### 권한 및 소유자

| :---: | :---: | :---: |
| `chmod` | 파일이나 폴더의 읽기, 쓰기, 실행 권한을 변경 (Change Mode) |  |

### 네트워크 통신

| 명령어 | 설명 | 사용 예시 |
| :---: | :---: | :---: |
| `ping` | 특정 서버나 웹사이트가 살아있는지(인터넷 연결이 잘 되는지) 신호를 보내 확인 |  |
| `wget` | 웹(인터넷) 상에 있는 파일을 내 컴퓨터로 다운로드 |  |
| `curl` | 웹 주소로 데이터를 보내거나 가져올 때 씁니다. API 테스트나 설치 스크립트를 다운로드할 때 주로 씁니다. |  |
| `ifconfig` / `ip` | 내 컴퓨터의 IP 주소와 네트워크 인터페이스 정보를 확인 |  |

### 압축 및 아카이브 (묶기)

| 명령어 | 설명 | 사용 예시 |
| :---: | :---: | :---: |
| `tar` | 여러 파일을 하나의 파일로 묶어주거나(아카이브), 묶은 파일을 다시 풀어줍니 |  |
| `zip` / `unzip` | .zip 형식으로 압축하거나 압축을 해제 |  |
