우리는 일반적으로 터미널에서 Bash shell을 이용해 리눅스 OS와 소통한다.
<br>
보통은 리눅스 환경에서 터미널에서 치는 모든 명령어를 묶어서 리눅스 명령어라고 부르기는 하지만, 엄밀히 말하면 Bash 명령어와 리눅스 명령어는 다르다.

**Bash 명령어**는 shell이라는 프로그램 자체가 원래부터 할 줄 아는 내장 명령어이다. 반면, **리눅스 명령어**는 shell이 처리할 수 없는 복잡한 일들을 처리하기 위해 리눅스 OS 내부에 따로 설치해 둔 별도의 외부 프로그램들이다.

내가 지금 입력하는 명령어가 Bash 명령어인지, 리눅스 명령어인지 궁금하다면 터미널에 `type {명령어}`를 입력해서 확인할 수 있다.

```bash
user@server:~$ type cd
> cd is a shell builtin

user@server:~$ type cat
> cat is /usr/bin/cat
```

아래는 우리가 주로 리눅스 환경에서 사용하는 CLI 명령어들을 정리한 것이다.

### 파일 및 디렉토리 (폴더) 관리

| 명령어 | 설명 | 사용 예시 |
| :---: | :---: | :---: |
| `ls` | 현재 위치에 있는 파일과 폴더 목록 출력 (List) | - |
| `cd` | 다른 폴더로 이동 (Change Directory) | `cd {디렉토리}` |
| `pwd` | 현재 내가 있는 위치의 절대 경로 출력 <br> (Print Working Directory) | - |
| `mkdir` | 새로운 폴더 생성 (Make Directory) | `mkdir {폴더명}` |
| `rm` | 파일이나 폴더 삭제 (Remove) | - |
| `cp` | 파일이나 폴더 복사 (Copy) | 같은 폴더에 이름만 바꿔서 복사: `cp {복사할 파일} {새로 만들 파일}` <br> 파일을 다른 폴더로 복사: `cp {복사할 파일} {도착 폴더}` <br> 폴더를 통째로 복사: `cp -r {복사할 폴더} {새로 만들 폴더}` |
| `mv` | 파일을 다른 곳으로 이동 (Move) | `mv {이동할 파일} {도착 폴더}` |

- `ls -al`을 사용하면 숨김 파일까지 확인할 수 있다.

    ![fig1](Programming/Dev_Environment/Linux_CLI-1.png){: style="display:block; margin:0 auto; width:60%;"}
- `cd ..`는 상위 폴더, `cd ~`는 홈 디렉토리로 이동한다.
- 파일을 삭제할 때는 `rm {파일명}`, 폴더를 삭제할 때는 `rm -rf {폴더명}` 형태로 사용한다.
- 파일의 이름을 바꾸기 위해 `mv {기존 파일명} {바꿀 파일명}`처럼 제자리에서 위치만 옮기는 방식으로도 사용할 수 있다.

#### 디렉토리

| 경로 표현 | 명칭 | 의미 | 실제 경로 |
| :---: | :---: | :---: | :---: |
| `/data` | 절대 경로 | 시스템의 최상위 (Root) 바로 아래의 data 폴더 | `/data` |
| `~/data` | 홈 경로 | 현재 로그인한 사용자의 개인 공간 내의 data 폴더 | `/home/{사용자 이름}/data` |
| `./data` | 상대 경로 | 현재 내가 있는 위치를 기준으로 그 안의 data 폴더 | `{현재 폴더 경로}/data` |

- **Root (`/`)**는 파일 시스템의 최상위 지점으로, 모든 경로는 `/`에서 시작된다.

    `/bin`, `/etc`와 같은 시스템 운영에 필수적인 핵심 폴더들이 위치한다.
    
    일반 사용자는 이곳의 파일을 마음대로 수정하거나 삭제할 수 없으며, 수정이 필요할 때는 `sudo` 같은 관리자 권한이 반드시 필요하다.

- **Home (`~`)**은 사용자마다 할당된 개인 작업 공간이다.

    개인 문서, 프로젝트 코드, 개인 설정 파일 등이 저장되는 곳이다.
    
    관리자 권한 없이도 자유롭게 파일을 생성하고 지울 수 있으며, 다른 사용자는 내 홈 디렉토리 내에 함부로 들어올 수 없다.

![fig2](Programming/Dev_Environment/Linux_CLI-2.png){: style="display:block; margin:0 auto; width:80%;"}

위는 일반적인 서버의 구조를 표현한 것이다.
<br>
일반적으로 `/data` 폴더에는 무거운 데이터를, `/home` 폴더에는 사용자별 개인 폴더를 저장하는 구조로 운영된다.

또는 Anaconda처럼 용량이 큰 프로그램은, `/data`에 실제 파일을 두고 `/home`에서는 심볼릭 링크로 연결해 사용할 수도 있다.
<br>
즉, `/home/suno/anaconda`는 `/data/suno/anaconda`를 가리키는 **바로가기**인 것이다.

### 파일 내용 보기 및 편집 / 파일 검색

| 명령어 | 설명 | 사용 예시 |
| :---: | :---: | :---: |
| `cat` | 파일의 전체 내용을 화면에 출력 (Concatenate) | `cat {파일명}` |
| `head` | 파일의 맨 앞부분만 출력 | `head {파일명}` |
| `tail` | 파일의 맨 뒷부분만 출력 | `tail {파일명}` |
| `nano` | 터미널 내에서 사용할 수 있는 텍스트 편집기 | `nano {파일명}` |
| `grep` | 텍스트 파일 안에서 특정 단어가 포함된 줄을 찾아서 출력 | `grep {찾을 단어} {파일명}` |
| `find` | 컴퓨터 안에서 특정 이름이나 확장자를 가진 파일/폴더를 찾아서 출력 | `find {검색할 경로} {옵션} {검색 조건}` |

아래와 같은 `.txt` 파일이 있다고 가정한다.

```txt
[2023-10-26 09:00:01] INFO: System started successfully.
[2023-10-26 09:05:23] WARNING: High memory usage detected on CPU 1.
[2023-10-26 09:15:42] ERROR: Failed to connect to the database.
[2023-10-26 09:20:05] INFO: User 'admin' logged in from 192.168.0.1.
[2023-10-26 09:45:12] ERROR: Python script crashed unexpectedly.
[2023-10-26 10:00:00] INFO: Scheduled backup completed.
[2023-10-26 10:12:33] WARNING: Disk space is running low (85% used).
```

- `nano`를 이용하면 아래의 창으로 진입하게 된다.

    ![fig3](Programming/Dev_Environment/Linux_CLI-3.png){: style="display:block; margin:0 auto; width:60%;"}

    텍스트를 수정한 뒤 `Ctrl + O`를 누르면 변경 사항을 저장할 수 있고, 저장하지 않고 그대로 빠져나오려면 `Ctrl + X`를 누르면 된다.
- `grep`은 기본적으로 대문자와 소문자를 구분한다. 만약 대소문자 상관없이 단어를 검색하고 싶다면, `-i` 옵션을 추가하면 된다.

    ![fig4](Programming/Dev_Environment/Linux_CLI-4.png){: style="display:block; margin:0 auto; width:60%;"}
- `|` 기호를 이용하면 앞선 명령어의 실행 결과를 다음 명령어의 입력으로 넘겨줄 수 있는데, 주로 `grep`과 함께 사용된다.

    예를 들어 `cat file1.txt | grep "ERROR"` 형태로 조합하면, `cat`으로 출력한 파일 내용 중에서 `ERROR`라는 글자가 포함된 줄만 필터링해서 보여준다.

    또한 `ls -al | grep "ERROR"` 형태로 조합해서, `ls -al`로 출력한 파일 리스트 중에서 `ERROR`라는 글자가 포함된 파일명만 필터링해서 볼 수도 있다.

    ![fig5](Programming/Dev_Environment/Linux_CLI-5.png){: style="display:block; margin:0 auto; width:60%;"}
- `find {검색할 경로} -name {검색 조건}`으로 특정 이름이나 확장자를 가진 파일/폴더를 찾을 수 있다.

    ```bash
    # 현재 폴더와 하위 폴더에서 이름이 "server.log"인 파일 찾기
    find . -name "server.log"

    # 시스템 전체에서 확장자가 .txt인 모든 파일 찾기
    find / -name "*.txt"

    # 대소문자 구분 없이 .jpg 확장자를 가진 파일 찾기 (JPG, Jpg 등 모두 검색)
    find . -iname "*.jpg"
    ```
- `find {검색할 경로} -size {검색 조건}`으로 특정 용량을 기준으로 필터링할 수 있다.

    ```bash
    # 용량이 100MB 이상인 파일들만 찾기
    find . -size +100M

    # 용량이 10KB 이하인 파일들만 찾기
    find . -size -10k
    ```

### 시스템 및 프로세스 관리

| 명령어 | 설명 | 사용 예시 |
| :---: | :---: | :---: |
| `ps` | 현재 실행 중인 프로세스들의 목록 출력 <br> (Process Status) | - |
| `kill` | 프로그램이나 프로세스를 강제 종료 | `kill -9 {PID}` |
| `df` | HDD나 SSD의 전체 용량과 남은 여유 공간 출력 <br> (Disk Free) | `df {옵션} {디렉토리}` |
| `du` | 특정 폴더나 파일이 디스크 용량을 얼마나 차지하는지 출력 <br> (Disk Usage) | `du {옵션} {디렉토리}` |

- `df`와 `du`를 그냥 사용하면 용량이 Byte 단위로 표기된다.
    <br>
    이때 `-h` (human-readable) 옵션을 사용하면, K(킬로), M(메가), G(기가) 단위로 바꿔서 보여준다.
- `df`와 `du`를 그냥 사용하면 모든 파일 목록이 다 출력된다.
    <br>
    이때 `-s` (summary) 옵션을 사용하면, 총합만 요약해서 보여준다.

    ```bash
    # 현재 위치에 있는 모든 내용물의 총 용량 확인
    user@server:~$ du -sh .
    > 100G  .

    # 특정 폴더가 차지하는 용량 확인
    user@server:~$ du -sh /var/log
    > 50G   /var/log

    # 현재 위치에 있는 모든 폴더/파일의 용량을 각각 요약해서 출력
    user@server:~$ du -sh *
    > 80G   Train_Folder
      4.0K  test.py
      19G   Exp_Folder
    ```
- 결과를 용량 순으로 정렬하고 싶다면 `|`와 `sort` 명령어를 결합하여 `du -sh * | sort -h` 형태로 사용하기도 한다.

    ```bash
    user@server:~$ du -sh * | sort -h
    > 80G   Train_Folder
      19G   Exp_Folder
      4.0K  test.py
    ```

### 네트워크 통신

| 명령어 | 설명 | 사용 예시 |
| :---: | :---: | :---: |
| `ping` | 특정 서버나 웹사이트가 살아있는지 신호를 보내 확인 <br> (인터넷 연결이 잘 되는지) | `ping {도메인 이름/IP 주소}` |
| `wget` | 웹 상에 있는 파일을 내 컴퓨터로 다운로드 (Web Get) | `wget {웹 URL}` |
| `curl` | 웹 주소로 데이터를 보내거나 가져옴 (Client URL) | `curl {웹 URL}` |
| `ifconfig` / `ip` | 내 컴퓨터의 IP 주소와 네트워크 인터페이스 정보 확인 | - |

- `wget`은 인터넷의 파일을 내 컴퓨터로 다운로드하기 위해 사용한다.

    주로 무거운 설치 파일이나 압축 파일을 받을 때 사용한다.

    ```bash
    # 아나콘다 설치 파일을 내 컴퓨터로 다운로드 
    wget https://repo.anaconda.com/archive/Anaconda3-2023.09-0-Linux-x86_64.sh
    ```
- `curl`은 파일 다운로드도 가능하지만, 그보다는 웹 서버에 데이터를 보내고, 요청하고, 결과값을 받아오는 통신에 특화된 명령어다.

   따라서 명령어를 입력하면 파일로 저장하지 않고 결과물을 터미널 창에 출력하기 때문에, 주로 API 결과값을 화면에서 바로 확인할 때 사용한다.

    ```bash
    # ChatGPT API를 이용해 질문에 대한 답변 출력
    curl https://api.openai.com/v1/chat/completions \
        -H "Content-Type: application/json" \
        -H "Authorization: Bearer My_OPENAI_API_KEY" \
        -d '{
            "model": "gpt-3.5-turbo",
            "messages": [{"role": "user", "content": "지금 서울의 시간은?"}]
        }'
    ```

### 압축 및 아카이브 (묶기)

| 명령어 | 설명 | 사용 예시 |
| :---: | :---: | :---: |
| `tar` | 여러 파일을 하나의 파일로 묶거나, 묶은 파일을 해제 | `tar {옵션} {파일/폴더명} {도착 폴더}` |
| `zip` / `unzip` | .zip 형식으로 압축하거나 압축을 해제 | - |

- 보통 리눅스나 서버 환경끼리는 `tar.gz` 형식을 표준처럼 사용하고, 윈도우 사용자와 파일을 주고받을 때는 `zip` 형식을 주로 사용한다.
- 일반적으로 압축할 때는 `tar -cvzf`를, 압축 해제할 때는 `tar -xvzf`를 사용한다.

### 기타 명령어

| 명령어 | 설명 | 사용 예시 |
| :---: | :---: | :---: |
| `history` | 내가 지금까지 터미널에 입력했던 명령어 기록 | - |
| `alias` | 긴 명령어를 짧은 단축어로 만듦 | `alias {단축어}={'원래명령어'}` |
| `clear` | 터미널 화면을 깨끗하게 지움 | - |
| `echo` | 내가 입력한 글자를 터미널 화면에 그대로 출력 <br> (프로그래밍 언어의 `print()` 함수와 같은 역할) | `echo {텍스트}` |

- `alias` 사용법은 다음과 같다.

    ```bash
    # 'ls -al'을 매번 치기 귀찮을 때 'll'이라는 단축어 생성
    alias ll='ls -al'

    # 현재 설정된 모든 alias 목록 보기
    alias

    # 설정한 단축어 'll' 삭제
    unalias ll
    ```
- `echo`와 redirection 기호 `>`, `>>`를 함께 사용하면, `nano` 같은 편집기를 열지 않고도 밖에서 파일에 바로 글자를 밀어 넣을 수 있다.

    ```bash
    # 덮어쓰기 (>): hello.txt 파일에 "Hello World"를 적는다. (기존 내용이 있으면 지우고 덮어씀)
    echo "Hello World" > hello.txt

    # 이어쓰기 (>>): hello.txt 파일의 맨 아랫줄에 "Nice to meet you"를 추가한다.
    echo "Nice to meet you" >> hello.txt
    ```
